# 모듈 1. 심층 인터뷰 기법 (30분)

## 1.1 인터뷰의 목표

심층 인터뷰의 핵심은 고객이 **말하는 것(What they say)**이 아니라 **진짜 원하는 것(What they really need)**을 찾아내는 것이다.

**표면적 답변의 함정:**
- 고객: "스타벅스 커피가 맛있어서 매일 가요"
- 진짜 니즈: 지루한 출근길의 작은 위안, 일상의 루틴, 스트레스 해소

고객은 자신의 진짜 니즈를 스스로도 모르는 경우가 많다. 인터뷰어의 역할은 대화를 통해 그 숨겨진 니즈를 함께 발견하는 것이다.

---

## 1.2 래더링(Laddering) 기법

### 개념
표면적 답변에서 시작해 "왜?"를 반복하며 근본 동기까지 파고드는 기법이다. 단, "왜요?"를 계속 반복하면 취조처럼 느껴지므로 다양한 표현으로 바꿔가며 질문한다.

### 사례 A: 배달앱 사용자

| 단계 | 질문과 답변 |
|------|-------------|
| 표면 | "배달앱으로 주문할 때 항상 리뷰를 먼저 봐요" |
| Why 1 | "리뷰에서 주로 뭘 확인하세요?" → "맛있는지, 양은 괜찮은지요" |
| Why 2 | "사진도 많이 보시나요?" → "네, 사진이랑 실제가 다를까봐요" |
| Why 3 | "그런 경험이 있으셨어요?" → "몇 번 속아서 기분이 나빴어요" |
| Why 4 | "그때 어떤 기분이셨어요?" → "내 선택이 틀린 것 같고, 돈 낭비한 느낌이었어요" |
| **진짜 니즈** | 맛집 발견이 아니라 **"실패하지 않는 선택에 대한 확신"**이 필요 |

### 사례 B: 헬스장 등록 고객

| 단계 | 질문과 답변 |
|------|-------------|
| 표면 | "3개월 등록했는데 한 달 만에 안 가게 됐어요" |
| Why 1 | "어떤 상황이었나요?" → "퇴근 후에 가려고 했는데 시간이 없더라고요" |
| Why 2 | "퇴근하고 나면 어떠세요?" → "피곤해서 집에 가고 싶어져요" |
| Why 3 | "그때 어떤 생각이 드세요?" → "오늘 하루만 쉬자... 가 계속 반복됐어요" |
| Why 4 | "처음에 등록하신 이유는 뭐였어요?" → "건강검진에서 경고 받아서요" |
| **진짜 니즈** | 헬스장이 아니라 **"의지력 없이도 유지되는 건강관리 시스템"**이 필요 |

### 사례 C: 은행 앱 사용자

| 단계 | 질문과 답변 |
|------|-------------|
| 표면 | "은행 앱으로 이체할 때 항상 긴장돼요" |
| Why 1 | "어떤 부분이 긴장되세요?" → "실수할까봐요" |
| Why 2 | "어떤 실수요?" → "계좌번호 잘못 입력하면 큰일이잖아요" |
| Why 3 | "직접 경험하신 적 있으세요?" → "저는 없는데 주변에서 들었어요" |
| Why 4 | "그 이야기 들으셨을 때 어떠셨어요?" → "나도 그럴 수 있겠다 싶어서 무서웠어요" |
| **진짜 니즈** | 더 쉬운 UI가 아니라 **"되돌릴 수 있다는 안전장치"**가 필요 |

### "왜?"를 다르게 표현하는 방법

- "그때 어떤 생각이 드셨어요?"
- "좀 더 말씀해주시겠어요?"
- "그게 어떤 의미인가요?"
- "구체적으로 어떤 상황이었나요?"
- "그래서 어떻게 하셨어요?"

## 1.3 극단적 사용자(Extreme Users) 인터뷰

### 개념
평균적인 사용자가 아닌 극단적 상황의 사용자에게서 혁신 포인트를 발견하는 기법이다. 극단적 사용자가 겪는 문제는 일반 사용자도 약하게 겪고 있는 경우가 많다.

### 사례 A: OXO 주방용품 - 채소 껍질칼

| 극단적 사용자 | 발견한 인사이트 |
|---------------|-----------------|
| 관절염 환자 | 얇은 손잡이를 오래 잡기 어려움 |
| 전문 요리사 (헤비유저) | 장시간 사용 시 손에 물집 |
| 왼손잡이 | 기존 제품 대부분 오른손 전용 설계 |
| 어린이 요리교실 | 손이 작아서 미끄러짐 |

**결과:** 두껍고 미끄럼 방지되는 유니버설 디자인 손잡이 개발. 관절염 환자를 위한 디자인이 모든 사용자에게 편리한 제품이 됨.

### 사례 B: 나이키 러닝화 개발

| 극단적 사용자 | 발견한 인사이트 |
|---------------|-----------------|
| 울트라 마라토너 | 100km 이상 달리면 발톱이 빠짐 (앞코 공간 부족) |
| 초보 러너 | 무릎 통증 호소 (쿠셔닝 부족) |
| 트레일 러너 | 젖은 바위에서 미끄러짐 |
| 야간 러너 | 차에서 안 보여서 위험함 |

**결과:** 발가락 여유 공간 확보, 부위별 차등 쿠셔닝, 반사 소재 적용

### 사례 C: 통신사 고객센터 (가상)

| 극단적 사용자 | 발견한 인사이트 |
|---------------|-----------------|
| 청각장애인 | 전화 상담 자체가 불가능 |
| 고령층 (70대 이상) | ARS 메뉴 기억 못함, 앱 사용 어려움 |
| 해외 체류 고객 | 시차 때문에 상담 시간 맞추기 어려움 |
| 민원 전문가 (월 10회 이상 문의) | 매번 같은 정보 반복 설명에 지침 |

**도출된 질문:** "어떻게 하면 전화 없이도 복잡한 문제를 해결할 수 있을까?"

## 1.4 JTBD(Jobs to Be Done) 기법

### 개념
고객이 제품/서비스를 "고용"해서 완수하려는 진짜 일(Job)을 찾는 기법이다. 고객은 제품 자체가 아니라 그 제품이 해결해주는 "일"을 구매한다.

### 사례 A: 밀크쉐이크 (클레이튼 크리스텐슨 연구)

패스트푸드점에서 밀크쉐이크 매출을 늘리고 싶어 고객 조사를 진행했다.

| 조사 내용 | 발견 |
|-----------|------|
| 언제 구매하나? | 아침 출근길 드라이브스루에서 |
| 왜 구매하나? | 운전하면서 한 손으로 먹기 좋음 |
| 경쟁자는 누구인가? | 바나나, 베이글, 도넛 (다른 음료가 아님!) |
| **진짜 Job** | "지루한 출근길을 달래주면서 점심까지 배고프지 않게 해줘" |

**결과:** 더 걸쭉하게 만들어 오래 먹을 수 있게 함, 과일 조각 추가로 씹는 재미 제공

### 사례 B: 전동 드릴

| 고객이 말한 것 | 진짜 Job |
|----------------|----------|
| "좋은 드릴이 필요해요" | "벽에 깔끔한 구멍을 뚫고 싶어요" |
| "강력한 모터가 필요해요" | "선반을 안전하게 설치하고 싶어요" |
| **궁극적 Job** | "집을 예쁘게 꾸며서 가족에게 자랑하고 싶어요" |

### JTBD 인터뷰 질문

- "이 제품/서비스를 처음 이용하기로 결심한 순간을 기억하세요? 그때 상황이 어땠나요?"
- "이걸 이용하기 전에는 어떻게 해결하셨어요?"
- "이게 없어진다면 뭘로 대체하실 것 같아요?"
- "이걸 쓰면서 기대했던 것과 달랐던 점이 있나요?"

## 1.5 감정 여정(Emotional Journey) 기법

### 개념
서비스 경험 전체를 시간순으로 따라가며 감정의 변화를 추적한다. 감정이 급격히 떨어지는 지점이 핵심 개선 포인트다.

### 사례: 병원 진료 경험

| 단계 | 경험 | 감정 |
|------|------|------|
| 예약 | "앱으로 간편하게 예약했어요" | 😊 좋음 |
| 이동 | "주차할 데가 없어서 20분 헤맸어요" | 😤 나쁨 (페인포인트) |
| 접수 | "접수는 금방 됐어요" | 😐 보통 |
| 대기 | "40분 기다렸어요" | 😤 나쁨 (페인포인트) |
| 진료 | "선생님이 친절하게 설명해주셨어요" | 😊 좋음 |
| 수납 | "수납 줄이 또 길더라고요" | 😤 나쁨 (페인포인트) |
| 약국 | "처방전 들고 약국 찾아다녔어요" | 😩 매우 나쁨 (페인포인트) |

**도출된 개선 기회:** 주차 사전 안내, 대기 순번 실시간 알림, 원내 약국 연계

### 감정 여정 인터뷰 질문

- "처음부터 끝까지 순서대로 말씀해주시겠어요?"
- "그 순간에 기분이 어떠셨어요?"
- "가장 기분이 좋았던/나빴던 순간은 언제였나요?"
- "다시 그 상황이 되면 다르게 하실 부분이 있나요?"

## 1.6 워크어라운드(Workaround) 발견 기법

### 개념
고객이 불편함을 해결하기 위해 스스로 만든 임시방편에서 혁신 기회를 발견한다. 워크어라운드가 있다는 것은 해결되지 않은 니즈가 있다는 증거다.

### 워크어라운드 사례와 숨겨진 니즈

| 발견한 워크어라운드 | 숨겨진 니즈 | 솔루션 기회 |
|--------------------|-------------|-------------|
| 리모컨 버튼에 테이프로 "전원" 표시 | 버튼이 너무 많아 헷갈림 | 단순화된 시니어용 리모컨 |
| 냉장고에 유통기한 포스트잇 | 식재료 관리가 어려움 | 스마트 냉장고, 식재료 관리 앱 |
| 약 봉투에 요일을 매직으로 표시 | 복약 관리 어려움 | 복약 알림 앱, 요일별 약통 |
| 카톡 "나에게 보내기"로 메모 | 기존 메모앱이 불편함 | 더 접근성 좋은 퀵메모 |
| 주차 위치 사진 찍어두기 | 대형 주차장에서 길 잃음 | 주차 위치 자동 기록 |
| 고객센터 전화번호 즐겨찾기 저장 | 번호 찾기가 번거로움 | 앱 내 원터치 상담 연결 |

### 워크어라운드 발견 질문

- "이 문제를 나름대로 해결하는 본인만의 방법이 있으세요?"
- "불편하지만 그냥 참고 쓰는 것들이 있나요?"
- "친구나 가족에게 배운 꿀팁 같은 게 있으세요?"
- "이 서비스를 더 잘 쓰려고 따로 뭔가를 하시나요?"

## 1.7 인터뷰 황금 질문 리스트

### 경험 탐색 질문
- "가장 최근에 [서비스]를 이용했을 때 상황을 처음부터 끝까지 말씀해주세요"
- "그때 가장 기억에 남는 순간은 언제였나요?"
- "만약 마법 지팡이가 있다면 뭘 바꾸고 싶으세요?"

### 감정 탐색 질문
- "그때 기분이 어떠셨어요?"
- "그 상황에서 제일 불편했던 점은요?"
- "반대로 만족스러웠던 순간도 있었나요?"

### 행동 관찰 질문
- "실제로 어떻게 해결하셨어요?"
- "다른 방법을 시도해보신 적 있으세요?"
- "주변에 같은 고민을 가진 분이 계신가요? 그분은 어떻게 하시던가요?"

## 1.8 인터뷰 Do's & Don'ts

### 해야 할 것 (Do's)

| 항목 | 설명 |
|------|------|
| 열린 질문하기 | "말씀해주세요", "어떠셨어요?" 형태로 질문 |
| 침묵 허용하기 | 3초 정도 기다리면 더 깊은 답변이 나옴 |
| 구체적 에피소드 요청 | "최근에 그런 경험이 있으셨나요?" |
| "왜"를 다양하게 표현 | "어떤 이유에서요?", "그게 무슨 의미인가요?" |
| 메모하며 경청 | 키워드 위주로 적고 눈 마주침 유지 |
| 비언어적 신호 관찰 | 표정, 제스처, 말투 변화 기록 |

### 하지 말아야 할 것 (Don'ts)

| 항목 | 설명 |
|------|------|
| 닫힌 질문 | "좋으셨어요?" (네/아니오 답변 유도) |
| 답변 유도 | "불편하셨죠?" (본인 생각 주입) |
| 일반적 의견 묻기 | "보통 어떻게 생각하세요?" (구체성 없음) |
| "왜요?" 반복 | 취조하는 느낌을 줄 수 있음 |
| 솔루션 제안 | "이렇게 하면 되지 않나요?" (인터뷰 목적 벗어남) |
| 말 끊기 | 답변 중간에 끼어들기 |

# 모듈 2. AI 사전 리서치 실습 (40분)

## 2.1 왜 사전 리서치가 필요한가

**핵심 개념: AI 리서치는 숙제, 현장 인터뷰는 시험**

| 구분 | AI 리서치 (숙제) | 현장 인터뷰 (시험) |
|------|------------------|-------------------|
| 알 수 있는 것 | 무엇이(What) 문제인지 | 왜(Why) 그런지 |
| 정보의 성격 | 이미 공개된 정보 | 개인의 경험과 감정 |
| 데이터 형태 | 통계, 리뷰, 기사 | 스토리, 맥락, 뉘앙스 |
| 한계 | 평균적 불만만 파악 | 깊은 동기와 워크어라운드 발견 |

**숙제 안 하고 시험 보면?**
기본적인 것도 물어보느라 시간 낭비. "대기 시간이 긴 편인가요?" 같은 이미 알려진 정보 확인에 시간 소모.

**숙제만 하고 시험 안 보면?**
표면적인 정보만 알고 끝. "고객이 대기 시간에 불만이 있다"는 알지만, 그 시간 동안 무슨 생각을 하는지, 어떻게 대처하는지는 모름.

**숙제 + 시험 = 진짜 인사이트**
AI로 기본 정보를 파악하고, 인터뷰에서는 "왜", "어떻게", "그때 기분" 등 깊은 질문에 집중.

## 2.2 Step 1: 딥리서치로 기존 정보 수집 (15분)

### 실습 도구
- **Claude 리서치 기능** (추천)
- **Perplexity**
- **NotebookLM** (자료 정리용)

### 리서치 프롬프트 예시

```
SKT 대리점 고객 경험에 대해 조사해줘.

다음 내용을 중심으로 찾아줘:
1. SKT 대리점 관련 고객 불만 사례 (최근 1-2년)
2. 통신사 대리점에서 흔히 발생하는 고객 페인포인트
3. MZ세대와 시니어 고객의 대리점 이용 패턴 차이
4. 온라인 커뮤니티나 리뷰에서 자주 언급되는 이슈
5. 경쟁사(KT, LGU+) 대리점과 비교되는 점

가능하면 출처와 함께 정리해줘.
```

### 예상 리서치 결과 예시

| 카테고리 | AI로 찾은 정보 |
|----------|----------------|
| 공통 불만 | 대기 시간 김, 불필요한 부가서비스 권유, 요금제 설명이 복잡함, 약정 관련 불만 |
| MZ세대 특징 | 온라인으로 다 되는데 굳이 방문해야 하는 상황에 불만, 대면 상담 자체를 불편해함 |
| 시니어 특징 | 직원 설명이 너무 빠름, 전문용어 이해 어려움, 앱 설치나 설정에 도움 필요 |
| 긍정 경험 | 친절한 직원 응대, 복잡한 문제 즉석 해결, 신제품 체험 기회 |
| 경쟁사 비교 | 대리점 접근성, 프로모션 차이, 직원 전문성 |

3. 결과를 팀 공유 문서에 정리

## 2.3 Step 2: 리서치 결과 분류하기 (10분)

### 분류 프레임워크

**AI로 알 수 있는 것 (What)**
- 어떤 불만이 있는지
- 얼마나 자주 발생하는지
- 어떤 패턴이 있는지
- 용도: 검증/확인 질문으로 활용

**현장 인터뷰로만 알 수 있는 것 (Why/How/Feel)**
- 왜 그렇게 느끼는지
- 어떻게 대처하는지
- 그때 감정은 어땠는지
- 개인적인 맥락과 스토리
- 용도: 심층 질문으로 활용

### 실습 워크시트

| 리서치에서 찾은 이슈 | 이미 아는 것 (What) | 현장에서 물어볼 것 (Why/How/Feel) |
|---------------------|---------------------|----------------------------------|
| 대기 시간이 길다 | 평균 15-30분 대기 | "기다리는 동안 어떤 기분이셨어요?" "그 시간에 뭘 하셨어요?" "대기 시간을 줄이려고 해본 방법이 있나요?" |
| 부가서비스 권유 | 방문 시 권유 빈도 높음 | "그때 어떻게 대응하셨어요?" "거절하기 어려우셨나요?" "기분이 어떠셨어요?" |
| 요금제 설명 복잡 | 옵션이 많아 헷갈림 | "어떤 부분이 가장 헷갈리셨어요?" "결국 어떻게 결정하셨어요?" "나중에 후회하신 적 있나요?" |
| 온라인으로 해결 안 되는 문제 | 기기 개통, 복잡한 문의 등 | "온라인으로 먼저 시도해보셨나요?" "왜 결국 대리점에 오시게 됐나요?" |
| 시니어 사용 어려움 | 앱/전문용어 어려움 | "어떤 부분이 가장 어려우세요?" "누군가에게 도움 요청하시나요?" |

## 2.4 Step 3: 인터뷰 질문 리스트 생성 (10분)

### AI 활용 질문 생성 프롬프트

```
나는 SKT 대리점 고객 경험을 조사하는 인터뷰어야.

[리서치 결과]
- 고객들이 대기 시간, 부가서비스 권유, 복잡한 요금제에 불만이 있음
- MZ세대는 "온라인으로 다 되는데 굳이 방문해야 하나" 의문을 가짐
- 시니어는 설명 속도와 전문용어에 어려움을 느낌

이 정보를 바탕으로 인터뷰 질문을 만들어줘:

1. 검증용 질문 5개: 리서치 내용이 실제로 맞는지 간단히 확인하는 질문
2. 심층 탐색 질문 5개: 왜 그런지, 어떻게 느꼈는지, 구체적 경험을 묻는 질문
3. 워크어라운드 발견 질문 3개: 고객이 나름의 해결책을 찾았는지 확인하는 질문

디자인씽킹 인터뷰 기법(래더링, JTBD 등)을 적용해서 열린 질문으로 만들어줘.
```

### 예상 출력 예시

**검증용 질문**
1. 오늘 대리점 방문하시기 전에 대기 시간이 어느 정도일 거라고 예상하셨나요?
2. 방문 목적 외에 다른 서비스나 상품을 권유받으신 적이 있으신가요?
3. 요금제 선택하실 때 여러 옵션 중에서 고민이 되셨던 적 있으세요?
4. 대리점 오시기 전에 온라인이나 앱에서 먼저 해결해보려고 하셨나요?
5. 직원 분의 설명 속도나 용어가 이해하기 어려우셨던 적 있나요?

**심층 탐색 질문**
1. 대기하시는 동안 어떤 생각이 드셨어요? 그 시간을 어떻게 보내셨나요?
2. 서비스 권유받으셨을 때 솔직히 기분이 어떠셨어요? 어떻게 대응하셨나요?
3. 요금제 설명 들으실 때 "이건 뭐지?" 싶었던 순간이 있으셨다면 말씀해주세요.
4. 오늘 방문하시게 된 결정적인 계기가 있으셨나요? 그 전에 어떤 시도를 해보셨어요?
5. 대리점 경험 중에서 가장 기분이 좋았던 순간과 안 좋았던 순간을 말씀해주신다면요?

**워크어라운드 발견 질문**
1. 대리점 방문 전에 미리 알아보고 오시는 편인가요? 어떤 방법으로요?
2. 통신 관련해서 어려운 일 있을 때 도움 요청하시는 분이 계신가요? 보통 어떻게 해결하세요?
3. 대리점 말고 다른 방법으로 해결하신 적 있으세요? 어떤 경우에요?

- 메인이 놓친 "왜?" 질문 보완
- 인터뷰이가 메인에게만 집중하도록 개입 최소화
- 마지막에 "제가 추가로 여쭤봐도 될까요?" 형태로 참여

**기록자**
- 눈 마주침 최소화 (메모에 집중)
- 키워드만 빠르게, 문장으로 적지 않기
- 비언어적 신호 (표정, 목소리 톤 변화) 반드시 기록

**시간/장비 관리자**
- 녹음 시작 여부 수시 확인
- 남은 시간 메인 인터뷰어에게 조용히 알리기 (손가락으로)
- 인터뷰 종료 후 즉시 녹음 파일 백업

# 모듈 4. 모의 인터뷰 & 마무리 (25분)

## 4.1 모의 인터뷰 실습 (15분)

### 진행 방식
- 팀별로 2개 조로 나눔 (2명 vs 2명)
- 한 조가 인터뷰어, 한 조가 고객 역할
- 5분 진행 후 역할 교대
- 피드백 공유

### 고객 역할 시나리오 카드

**시나리오 A: 20대 직장인**
```
- 요금제 변경하러 방문
- 온라인에서 해보려다 포기하고 옴
- 시간이 없어서 빨리 끝내고 싶음
- 부가서비스 권유받으면 짜증남
```

**시나리오 B: 60대 시니어**
```
- 휴대폰 기종 변경하러 방문
- 자녀가 추천한 기종이 있는데 이름을 모름
- 직원 설명이 빠르면 잘 못 알아들음
- 질문하기가 부끄러움
```

**시나리오 C: 30대 학부모**
```
- 자녀 휴대폰 개통하러 방문
- 요금 폭탄 걱정됨
- 자녀가 어떤 앱을 쓰는지 관리하고 싶음
- 매장에서 아이가 딴 데 가서 신경 쓰임
```

### 피드백 포인트

모의 인터뷰 후 서로 피드백할 때 확인할 점:
- 열린 질문을 했는가?
- 침묵을 허용했는가?
- "왜?"를 자연스럽게 물었는가?
- 답변을 유도하지 않았는가?
- 비언어적 신호를 놓치지 않았는가?

## 4.2 자주 하는 실수 & 대처법

| 실수 상황 | 대처법 |
|-----------|--------|
| 녹음 버튼 안 눌렀음 | 인터뷰 직후 기억나는 대로 즉시 메모. 팀원들과 함께 복기 |
| 인터뷰이가 말이 없음 | "예를 들어서..." 또는 구체적 상황 제시. "최근에 그런 적 있으셨나요?" |
| 인터뷰이가 말이 너무 많음 | "아, 그 부분 흥미로운데요, 잠시 [핵심 주제]로 돌아가면..." |
| 내 의견을 말하고 싶어짐 | 참기. 메모장에 적어두고 인터뷰 끝난 후 팀과 공유 |
| 예/아니오 답변만 나옴 | "그때 상황을 좀 더 구체적으로 말씀해주시겠어요?" |
| 시간 초과될 것 같음 | "시간이 얼마 안 남았는데요, 마지막으로 꼭 여쭤보고 싶은 게..." |
| 인터뷰이가 불편해 보임 | "혹시 불편하신 부분 있으세요? 편하게 말씀해주세요" |
| 중요한 답변인데 더 파고들어야 할 것 같음 | "그 이야기 조금 더 해주시겠어요? 정말 흥미로워요" |

## 4.3 인터뷰 후 처리 플로우

### 즉시 (인터뷰 종료 직후)

1. 녹음 파일 저장 확인
2. 클라우드 백업 (카톡 나에게 보내기 또는 드라이브)
3. 5분 내 핵심 인상 메모 작성
4. 팀원과 짧은 디브리핑 (3분)
   - "나는 이 부분이 인상적이었어"
   - "나는 다르게 들었는데..."

### 당일 저녁

1. 녹음 파일 → 클로바노트 전사
2. 전사록 검토 및 오류 수정
3. NotebookLM에 업로드

### NotebookLM 활용 질문 예시

```
이 인터뷰에서:
- 가장 강한 불만이나 니즈는 뭐야?
- 인터뷰이가 언급한 워크어라운드가 있어?
- 감정이 가장 격해진 순간은 언제야?
- 이 인터뷰를 바탕으로 HMW 질문 3개 만들어줘
```

### 팀 종합 분석 (여러 인터뷰 완료 후)

```
우리 팀이 진행한 3개 인터뷰 전사록이야.
- 공통으로 나온 불만은 뭐야?
- 가장 자주 언급된 키워드는?
- 각 인터뷰에서만 나온 독특한 인사이트는?
- 이걸 바탕으로 핵심 페르소나를 만들어줘
```

## 4.4 최종 점검 & Q&A

### 출발 전 최종 체크

| 항목 | 확인 |
|------|------|
| 팀별 질문 리스트 완성 | ☐ |
| 역할 분담 확정 | ☐ |
| 장비 점검 완료 | ☐ |
| 녹음 동의 스크립트 숙지 | ☐ |
| 비상 연락망 공유 | ☐ |
| 집합 시간/장소 확인 | ☐ |

### 기억할 핵심 3가지

1. **숙제(AI 리서치)를 했으니, 시험(인터뷰)에서는 "왜"와 "어떻게"에 집중하기**

2. **침묵은 적이 아니다. 3초 기다리면 더 깊은 이야기가 나온다**

3. **완벽한 인터뷰는 없다. 실수해도 괜찮다. 기록만 잘 하면 된다**

## 부록: 유용한 프롬프트 모음

### 인터뷰 전 - 리서치용

```
[주제]에 대한 고객 경험을 조사해줘.
1. 주요 불만 사례
2. 연령대별 차이
3. 온라인에서 자주 언급되는 이슈
4. 경쟁사와 비교되는 점
출처와 함께 정리해줘.
```

### 인터뷰 전 - 질문 생성용

```
나는 [주제] 고객 경험을 조사하는 인터뷰어야.

[리서치 결과 요약 붙여넣기]

이 정보를 바탕으로:
1. 검증용 질문 5개
2. 심층 탐색 질문 5개 (왜/어떻게/감정)
3. 워크어라운드 발견 질문 3개

디자인씽킹 인터뷰 기법을 적용해서 열린 질문으로 만들어줘.
```

### 인터뷰 후 - 분석용

```
이 인터뷰 전사록을 분석해줘.

1. 핵심 인사이트 3가지
2. 가장 강한 페인포인트
3. 발견된 워크어라운드
4. 감정이 격해진 순간과 그 맥락
5. 추가로 탐구할 질문
```

### 종합 분석용

```
우리 팀이 진행한 [N]개 인터뷰 전사록이야.

1. 공통 불만과 니즈
2. 자주 언급된 키워드
3. 각 인터뷰 고유의 인사이트
4. 종합 페르소나 제안
5. HMW(How Might We) 질문 5개
```
